## Installed Skills (预加载记忆)

开发任务启动时自动加载以下 Skill 的核心规则：

### dev-workflow（标准化开发流程）

所有开发任务必须遵循 7 步流程：需求确认 → 建立测试 → 需求实现 → 测试验证 → 文档维护 → Review → 发布确认。
- Bug 修复使用专用流程：BF-1 全链路追踪 → BF-2 复现测试 → BF-3 影响分析 → BF-4 端到端验证
- 测试前置：先写测试（功能测试必须 FAIL），再实现（PASS）
- 必须全量影响面扫描（grep 所有使用方，P0/P1 一并修复）
- 禁止手动部署，只用 `scripts/deploy.sh`

### code-standards-check（开发规范检查）

`/code-standards-check` 执行 7 模块自动检查：文件组织、API Route、Store、前端组件、i18n、安全、模块拆分。
- 超大文件阈值：400 行警告、800 行阻塞
- Store 规则：`createAsync`/`updateAsync` 用 API 返回值更新本地，`deleteAsync` 先 await 再移除
- 前端规则：编辑防抖 500ms、内联编辑防 Enter/Blur 双触发、派生列表 useMemo、useEffect 依赖精确

### knowledge-crystallizer（知识结晶化）

`/knowledge-crystallizer` 从 Bug 日志提取经验模式，自动生成/更新 Rule。
- Rule 生成阈值：同一模式 ≥2 次建议、同一模块 ≥3 次建议、白屏/服务不可用强制
- 知识闭环：Bug 发生 → `bug-knowledge-accumulation` 自动记录 → 本 Skill 分析结晶 → Rule 自动加载

### pre-release-review（发布前检查）

`/pre-release-review` 执行 10 阶段检测：构建 → 编码规范 → 安全 → API → Store → 前端 → i18n → 技术债 → 双模式兼容 → 报告。
- 构建失败则阻塞发布
- 问题分级：CRITICAL（阻塞）/ HIGH（建议修复）/ MEDIUM（可延后）/ LOW（不阻塞）
- 紧急发布快速 Review：Phase 1+3+9+4 必做，其他可延后

## Architecture

### Overview
TeamClaw is an AI Agent management platform (v1.0.1) serving as an enhanced frontend for **OpenClaw Gateway**. Built with Next.js 14 App Router + TypeScript strict mode. It uses a **dual data source architecture**: local SQLite for project data (tasks, docs, members) and WebSocket for Gateway data (agents, sessions, cron, skills).

### Data Flow (4 paths)

1. **User → SQLite**: React component → Domain Store → `@/shared/lib/data-service.ts` (fetch) → `app/api/*` Route Handlers → Drizzle ORM → SQLite (`data/teamclaw.db`)
2. **User → Gateway**: React component → Domain Store → `@/shared/lib/gateway-client.ts` (WebSocket, Protocol v0.9.8) → OpenClaw Gateway
3. **Real-time push**: API write triggers `eventBus.emit()` (`@/shared/lib/event-bus.ts`) → SSE stream at `/api/sse` → `DataProvider` component auto-refreshes stores
4. **AI MCP commands**: AI Agent → `/api/mcp` (or `/api/mcp/external` with Bearer token) → `src/core/mcp/` executor → DB write + SSE broadcast → frontend refresh

### Database Layer (`src/core/db/`)

- **`src/core/db/schema.ts`**: 21 Drizzle ORM tables. All primary keys are Base58 short IDs (not UUID), generated by API layer. Timestamps use `integer` with `mode: 'timestamp'`. JSON fields use `text` with `mode: 'json'` + `$type<T>()`. Every table exports both `Select` and `Insert` types.
- **`src/core/db/index.ts`**: Singleton connection via `globalThis` (HMR safe). Enables WAL mode, foreign keys, 64MB cache, 5s busy timeout. Includes auto-migration logic that detects missing tables/columns and adds them incrementally—never manually edit `data/teamclaw.db`.

### Gateway Client (`src/shared/lib/gateway-client.ts`)

WebSocket client implementing OpenClaw Protocol v0.9.8. Handshake: challenge → connect (token + role `operator`) → `hello-ok`. Request/response model with 30s timeout. Auto-reconnect with exponential backoff (1s–60s, max 10 retries). Global singleton via `createGatewayClient()`/`getGatewayClient()`. Covers: snapshot, agents (CRUD + files), sessions, cron, skills, config, chat.

### API Routes (`app/api/`)

21 API modules, all RESTful. Key conventions:
- Collection routes: `route.ts` → GET (list) + POST (create)
- Resource routes: `[id]/route.ts` → GET + PUT + DELETE
- Dynamic params: `{ params }: { params: Promise<{ id: string }> }` with `await params`
- PUT/DELETE must check resource existence (404 if missing)
- Sensitive fields (e.g. `openclawApiToken`) must pass through `@/shared/lib/sanitize.ts` before response
- PUT uses `allowedFields` whitelist; cascading deletes use `db.transaction()`
- Error format: `{ error: string }` with appropriate HTTP status

### MCP System (`src/core/mcp/`)

Three-layer architecture for AI Agent tool calls:
- **`types.ts`**: 27 `ActionInstruction` types + `ExecutionResult` + `PendingQuestion`
- **`definitions.ts`**: 37 MCP tool JSON Schema definitions (name, description, parameters)
- **`executor.ts`**: Switch-case dispatcher; calls `/api/mcp` internally, refreshes Zustand stores on success
- **`src/domains/*/mcp.ts`**: Domain handlers (task, document, member, project, schedule, status, delivery, template, sop)

To add a new MCP tool: (1) register definition in `definitions.ts`, (2) add handler in `src/domains/*/mcp.ts`, (3) add switch-case in `executor.ts`.

### Domain Store Layer (`src/domains/*/store.ts`)

16 Zustand stores organized by domain, located in `src/domains/*/`. `useDataInitializer()` hook loads all data in parallel via `Promise.allSettled` on app startup.

Critical rules:
- `updateAsync`/`createAsync` must update local state with **API response data**, not the request body
- `deleteAsync` must **await** API success before removing from local state
- Success operations must `set({ error: null })`

### Frontend Components (`src/features/` / `src/shared/`)

- **Layout** (`src/shared/layout/`): `AppShell`, `Header`, `Sidebar`, `ThemeProvider`, `DataProvider`
- **Features** (`src/features/`): `agent-manager/`, `chat-panel/`, `document-editor/`, `sop-engine/`, `task-board/`, `wiki-editor/`, `skill-manager/`, `milestone-tracker/`, `settings/`, `landing/`
- **UI**: `src/shared/ui/` has shadcn/ui primitives—must use these, never re-implement
- **Editor**: `src/shared/editor/` has MarkdownEditor, HtmlPreview, etc.

### Middleware (`middleware.ts`)

Applies only to `/api/*`. Adds security headers, CORS (localhost in dev, `ALLOWED_ORIGINS` in prod), CSRF protection for mutating requests (Origin/Referer check), 1MB body limit, and `X-Request-Id` UUID per request.

### Internationalization (`src/shared/lib/i18n.ts`)

All user-visible text must use `react-i18next` `t()` function—no hardcoded strings. Single `translation` namespace with nested keys by module. Dual language: `en` + `zh`. Language detection: localStorage → browser → fallback `en`. Translated constants must be defined inside components after `useTranslation()`.

### Key Patterns

- **Path alias**: `@/*` maps to project root; no cross-layer relative imports
- **Input debounce**: All edit inputs must debounce 500ms via `useRef<setTimeout>`
- **Enter/Blur guard**: Inline edits use `submittedByEnterRef` to prevent double-submit
- **Derived lists**: Always `useMemo` for filter/map/sort operations
- **useEffect deps**: Must be specific fields (`[task?.id]`), never whole objects (`[task]`)
- **Dark mode**: `darkMode: 'class'` via `ThemeProvider`; all components must support both themes
- **CSS**: Tailwind only, no `<style jsx>`; theme via CSS variables (`--background`, `--text-primary`, etc.)
- **Icons**: `lucide-react` exclusively; conditional classNames via `clsx()`
- **Comments**: Write code comments in Chinese
