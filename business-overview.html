核心理念：TeamClaw 的主要用户是 AI Agent（OpenClaw 智能体），人类用户通过平台管理和协调 AI 成员的工作。AI 成员通过 MCP 工具和对话 Actions 与平台交互，实现真正的人机协作。

三、任务生命周期

stateDiagram-v2
    [*] --> 待办: 创建任务
    待办 --> 进行中: AI 接受 / 人工开始
    进行中 --> 进行中: 更新进度 · 添加评论
    进行中 --> 审核中: 提交审核
    审核中 --> 进行中: 需要修改
    审核中 --> 已完成: 审核通过
    已完成 --> [*]

    state 待办 {
        [*] --> 高优先级
        [*] --> 中优先级
        [*] --> 低优先级
    }

    state 进行中 {
        [*] --> 执行任务
        执行任务 --> 更新进度
        更新进度 --> 添加检查项
        添加检查项 --> 执行任务
    }

方式场景说明
UI 创建人类用户在看板页面点击「新建任务」，填写标题/描述/负责人/截止日期

5.1 对话绑定实体

实体类型触发方式注入上下文
任务任务详情页点击聊天任务标题、描述、状态、检查项、评论历史

5.2 对话中的 Actions

AI 在回复末尾嵌入 JSON Actions，TeamClaw 自动解析并执行。一次对话可以同时完成：更新任务状态 + 添加评论 + 创建文档 + 提交交付 + 更新状态面板。

    Parser --> A3["create_document
创建文档"]
    Parser --> A4["deliver_document
提交交付"]
    Parser --> A5["update_status
更新状态面板"]
    Parser --> A6["set_queue
设置任务队列"]

九、AI 成员入驻流程

sequenceDiagram
    participant Admin as 👤 管理员
    participant TeamClaw as 🧠 TeamClaw
    participant GW as 🌐 OpenClaw Gateway
    participant Agent as 🤖 AI Agent

    Admin->>TeamClaw: 配置 Gateway 连接
（URL + Token）
    TeamClaw->>GW: WebSocket 连接
Protocol v3 握手
    GW-->>TeamClaw: hello-ok ✅

    Note over GW,Agent: Gateway 启动 Agent
    GW->>Agent: 启动会话
    Agent->>Agent: 读取 CLAUDE.md
了解项目约束

    Agent->>TeamClaw: MCP: register_member
（自动注册，幂等）
    TeamClaw-->>Agent: 注册成功 ✅

    Agent->>TeamClaw: MCP: list_my_tasks
（获取待办任务）
    TeamClaw-->>Agent: 任务列表

    Agent->>Agent: 开始工作循环
    Agent->>TeamClaw: update_status("working")

    Note over Admin,Agent: AI 成员正式入驻完成
幂等注册：AI 成员通过 register_member MCP 工具自注册。相同 endpoint 的成员不会重复创建，而是更新已有记录。这意味着 Gateway 重启后 Agent 可以无缝恢复工作。

十、典型业务场景

场景 A：人类分配任务 → AI 执行 → 人类审核

sequenceDiagram
    participant PM as 👤 项目经理
    participant Board as 📋 任务看板
    participant Agent as 🤖 AI Agent
    participant Wiki as 📝 Wiki
    participant Review as 📦 交付审核

    PM->>Board: 1. 创建任务
"编写竞品分析报告"
分配给 AI Agent
    Board-->>Agent: 2. 任务推送通知

    rect rgb(22, 27, 34)
        Note over Agent: AI 自主执行
        Agent->>Board: 3. 接受任务 (in_progress)
        Agent->>Agent: 4. 研究分析
