### 1.2 缺失项（需补齐）

#### P1：节点配置面板（右侧属性面板）

**现状**：选中节点后只能编辑 label（双击），无法配置节点属性。

**需要新增**：在 `app/workflows/[id]/page.tsx` 的 Nodes Tab 中，选中节点时右侧显示属性面板：

| 节点类型 | 可配置属性 | 说明 |
|---------|-----------|------|
| `sop` | `sopTemplateId` | 从已有 SOP 模板列表中选择关联 |
| `ai_auto` | `description`, `promptTemplate` | AI 自动节点的提示词 |
| `input` | `description` | 输入节点的说明文本 |
| `render` | `description` | 渲染节点的说明 |
| `condition` | `condition.trueNext`, `condition.falseNext` | 条件分支的目标节点 + 条件表达式 |
| `loop` | `loop.maxIterations`, `loop.exitCondition` | 循环次数和退出条件 |
| `parallel` | `parallel.branches` | 并行分支列表 |
| `workflow_call` | `workflowId` | 子流程选择 |

**实现方式**：
- 详情页改为三栏布局（左：Canvas/列表 | 中：节点画布 | 右：属性面板）
- 属性面板根据 `selectedNodeId` 对应的节点类型动态渲染表单
- 修改直接调用 `handleUpdateNodeConfig(nodeId, partial)` 更新 nodes JSON

**文件变更**：
1. `app/workflows/[id]/page.tsx` — 增加属性面板 JSX（~100 行）
2. `src/core/workflow/types.ts` — 确认 `WorkflowNode.condition` / `.parallel` / `.loop` 字段定义

#### P2：发布/归档操作按钮

**现状**：列表页和详情页都**没有**状态切换按钮，workflow 创建后永远停留在 `draft` 状态。

**需要新增**：
- 列表页（`app/workflows/page.tsx`）：每行卡片增加 Publish/Archive 按钮
- 详情页（`app/workflows/[id]/page.tsx`）：头部增加状态切换下拉
- `useWorkflowStore.updateWorkflowAsync(id, { status: 'published' })` 已存在，直接调用

**文件变更**：
1. `app/workflows/page.tsx` — 卡片操作菜单增加 Publish/Archive（~20 行）
2. `app/workflows/[id]/page.tsx` — 头部增加状态 Badge + 切换按钮（~20 行）

#### P3：Run 详情视图

**现状**：Runs Tab 只显示简单的状态行（ID + status + currentNodeId），无节点级别的执行时间线。

**需要新增**：点击某个 Run 后展示节点执行详情：
- 每个已执行节点的状态（pending/running/completed/failed）
- 执行时间（startedAt → completedAt）
- 节点输出数据（如果有的话）

**注意**：当前 `workflowRuns.nodeHistory` 是 JSON 数组，已有数据。但**没有 `workflow_run_nodes` 表**（schema 中不存在），无法追踪每个节点的独立开始/结束时间。当前实现只能从 `nodeHistory` 中提取数据。

**文件变更**：
1. `app/workflows/[id]/page.tsx` — Runs Tab 增加点击展开详情（~80 行）

## 四、代码规范提醒

1. **API 错误消息必须使用英文**（前端通过 `t()` 翻译）
2. **编辑输入必须防抖 500ms**
3. **内联编辑用 ref 防 Enter/Blur 双重提交**
4. **Store `createAsync`/`updateAsync` 用 API 返回值更新本地状态**
5. **所有用户可见文本用 `t()` 函数，禁止硬编码**
6. **PUT/DELETE 必须校验资源存在性，不存在返回 404**
