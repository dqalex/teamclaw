# TeamClaw V3.0 正式发布：多用户系统与 SOP 工作流引擎

> 发布日期：2026-03-07  
> 版本：v3.0.0  
> 标签：重大更新, 多用户, SOP引擎

---

## 概述

TeamClaw V3.0 是一次重大的架构升级，带来了**多用户认证系统**和**SOP（标准化操作流程）工作流引擎**两大核心功能。从个人工具正式进化为团队协作平台，让 AI Agent 真正成为企业工作流中的团队成员。

---

## 核心新特性

### 1. 多用户认证系统（Multi-User System）

V3.0 引入了完整的企业级用户管理体系：

| 特性 | 说明 |
|------|------|
| **三角色权限** | Admin（管理员）、Member（成员）、Viewer（只读） |
| **安全码验证** | 敏感操作需二次验证，增强安全性 |
| **用户注册** | 支持开放注册和邀请制两种模式 |
| **用户隔离** | 聊天会话、个人配置完全隔离 |

**使用场景**：
- 企业团队多人协作，不同成员拥有不同权限
- 管理员统一管理 Gateway 配置和系统设置
- 普通成员专注于任务和项目协作

---

### 2. SOP 工作流引擎（Standard Operating Procedure）

让 AI Agent 按预定义的多阶段工作流自动执行复杂任务。

#### 7 种阶段类型

| 阶段类型 | 说明 |
|----------|------|
| `input` | 人工输入阶段，收集用户表单数据 |
| `ai_auto` | AI 自动执行，完成后自动推进 |
| `ai_with_confirm` | AI 执行后需人工确认再推进 |
| `manual` | 纯人工操作阶段 |
| `render` | 可视化渲染阶段（自动创建 Content Studio 文档） |
| `export` | 导出阶段 |
| `review` | 最终审核阶段 |

#### L1-L5 知识库分层

SOP 模板可关联 Know-how 文档，按五层结构组织：

- **L1 概述**：背景和目标
- **L2 核心知识**：关键方法论
- **L3 工具方法**：具体工具使用
- **L4 经验教训**：历史经验（AI 可自动追加）
- **L5 统计数据**：执行统计

#### 内置模板

| 模板 | 阶段数 | 用途 |
|------|--------|------|
| 竞品调研 SOP | 5 | 竞品信息收集→分析→报告 |
| 内容营销 SOP | 6 | 选题→创作→审核→发布 |
| 周报月报 SOP | 4 | 数据收集→汇总→渲染→审核 |
| Bug 分析 SOP | 5 | 复现→根因→修复→验证→知识沉淀 |
| 数据分析 SOP | 5 | 需求→采集→分析→可视化→审核 |

### 3. 首页内容管理系统

V3.0 新增可编辑的落地页内容管理：

- **可视化编辑**：通过 Settings 页面直接编辑首页内容
- **多语言支持**：中英文版本独立管理
- **模板系统**：支持 MD ↔ HTML 双向同步
- **实时预览**：所见即所得的编辑体验

### 4. 图标系统升级

内置 **Lucide Icons** 支持，所有模板均可使用：

```markdown

  四列看板 + 拖拽排序...
```

支持 120+ 常用图标，自动渲染为 SVG。

### 5. 博客系统

V3.0 新增博客功能，复用 Wiki 文档系统：

- **类型标记**：Wiki 中创建类型为 `blog` 的文档
- **自动同步**：自动显示在 `/blog` 页面
- **时间排序**：按更新时间倒序排列
- **标签支持**：支持文档标签分类

## 升级指南

### 从 V2 升级到 V3

1. **备份数据**：复制 `data/teamclaw.db` 到安全位置
2. **执行迁移**：启动 V3 时自动执行数据库迁移
3. **初始化**：首次启动需要创建管理员账户
4. **配置 Gateway**：重新配置 OpenClaw Gateway 连接

### 数据库迁移

V3 自动检测并迁移：

- 用户表（users）- 新增
- 项目成员表（project_members）- 新增
- 活动日志表（activity_logs）- 新增
- 文档表 - 新增 `blog` 类型
- 任务表 - 新增 SOP 相关字段

## 最佳实践

### 团队协作流程

```
1. 管理员创建项目并分配成员
2. 成员在项目中创建任务
3. 为任务关联 SOP 模板
4. AI 按 SOP 阶段自动执行
5. 人工在关键节点确认推进
6. 完成产出交付物
```

### SOP 设计原则

1. **阶段粒度适中**：每个阶段应有明确产出
2. **人机结合**：AI 擅长的用 `ai_auto`，需要判断的用 `ai_with_confirm`
3. **知识沉淀**：每个 SOP 都关联 Know-how 文档
4. **持续优化**：基于 L4 经验数据迭代模板

## V3.1 新增特性

### Skill 管理系统

V3.0 引入了完整的 AI Skill 生命周期管理：

| 特性 | 说明 |
|------|------|
| **Skill 注册** | 自动验证 SKILL.md 结构规范 |
| **审批流程** | Skill 发布需管理员审批 |
| **信任管理** | 信任/拒绝未知来源 Skill |
| **快照监控** | 定期检测 Agent 已安装 Skill 变更 |
| **敏感检测** | 自动标记含敏感信息的 Skill |

**Skill 状态流转**：
```
draft → pending_approval → active / rejected
```

**Skill 来源类型**：
- `teamclaw` - 内置 Skill
- `manual` - 手动创建
- `external` - 外部导入
- `unknown` - 未知来源

---

### 通用审批系统

统一的多场景审批流程：

**4 种审批类型**：
| 类型 | 说明 |
|------|------|
| `skill_publish` | Skill 发布审批 |
| `skill_install` | Skill 安装审批 |
| `project_join` | 项目加入申请 |
| `sensitive_action` | 敏感操作审批（预留） |

**审批策略配置**：
- 审批人规则：`any_admin` / `specific_role` / `project_admin`
- 超时设置：支持自动处理（批准/拒绝）
- 通知配置：申请人/审批人通知

**权限矩阵**：
| 操作 | 申请人 | 管理员 | 其他用户 |
|------|--------|--------|----------|
| 创建审批请求 | ✅ | ✅ | ✅ |
| 查看所有请求 | ❌ | ✅ | ❌ |
| 批准/拒绝请求 | ❌ | ✅ | ❌ |
| 取消自己的请求 | ✅ | ✅ | ❌ |
